# Projects

Directory containing all personal projects.

## Structure

- **frontend/**: Pure frontend projects (React, Vue, Svelte, etc.)
- **backend/**: Backend projects (API services, microservices)
- **fullstack/**: Fullstack projects (Next.js, Remix, etc.)
- **mobile/**: Mobile applications (React Native, Flutter, Native)
- **desktop/**: Desktop applications (Electron, Tauri, etc.)
- **ml-ai/**: Machine Learning and AI projects

## Naming Conventions

- Use kebab-case: `my-awesome-project`
- Names should clearly describe the project's purpose
- Avoid overly generic names like `test`, `project1`

## Template for Each Project

Each project should have:

- `README.md`: Project description, setup and run instructions
- `package.json` / `requirements.txt` / `Cargo.toml`: Dependencies
- `.env.example`: Template for environment variables
- `docker-compose.yml` (if needed): Local development setup
- Tests: Unit tests and integration tests
- Documentation: API docs, architecture diagrams
