# Learnings

Notes and insights from the learning and project-building process.

## Structure

Organized by topics or projects:

- `architecture/`: Architecture and design patterns
- `frontend/`: Frontend technologies and patterns
- `backend/`: Backend technologies and patterns
- `devops/`: DevOps and infrastructure
- `algorithms/`: Algorithms and data structures
- `best-practices/`: Best practices and tips

## Format

Each learning note should have:

- **Title**: Name of the concept/pattern
- **Date**: When it was learned
- **Context**: In which project
- **What**: What the concept is
- **Why**: Why it's important
- **How**: How to implement
- **Examples**: Code examples
- **Resources**: Links and references

## Example

```markdown
# Clean Architecture

**Date**: 2024-01-15
**Project**: [Project Name]

## What

Clean Architecture is a way to organize code...

## Why

- Separation of concerns
- Testability
- Independence from frameworks

## How

[Implementation details]

## Examples

\`\`\`typescript
// Code example
\`\`\`

## Resources

- [Link 1](url)
- [Link 2](url)
```
