# Documentation

Documentation and knowledge from projects.

## Structure

### architecture/

Architecture Decision Records (ADRs) and design documents:

- System design decisions
- Technology choices
- Pattern implementations
- Trade-offs analysis

### guides/

How-to guides and tutorials:

- Development setup
- Deployment procedures
- Debugging tips
- Best practices

### learnings/

Notes and insights from the learning process:

- Concepts learned
- Mistakes and solutions
- Resources and references
- Code snippets and examples

## Writing Guidelines

1. **Clear and concise**: Write clearly and easy to understand
2. **Examples**: Always include code examples
3. **Diagrams**: Use diagrams when needed (Mermaid, PlantUML)
4. **Links**: Link to resources and references
5. **Updates**: Update when changes occur

## Format

- Markdown for text documents
- Mermaid for diagrams
- Code blocks with syntax highlighting
